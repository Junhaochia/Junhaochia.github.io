<p>The string <i>type</i> is a sequence of characters.
</p><p>The string <i>library</i> is a standard Lua library which provides functions for the manipulation of strings.<a rel="nofollow" class="external autonumber" href="https://www.lua.org/pil/20.html">[1]</a>
</p><p>In Garry's Mod there are several extra useful functions and features added to this library.<br />
Most notably all strings will access this library through the string metatable index function.<a rel="nofollow" class="external autonumber" href="https://github.com/Facepunch/garrysmod/blob/master/garrysmod/lua/includes/extensions/string.lua#L288-L299">[2]</a>
</p><p>This means all strings are treated like table objects and the string library as its <a rel="nofollow" class="external text" href="/page/Meta_Tables">Meta Table</a>
</p>
<pre><code data-language="lua">local x = &quot;Kittens&quot;
function string.Foobar(self) return self:Right(4) end

string.Right(x,4) == x:Right(4) == x:Foobar() == (&quot;Kittens&quot;):Right(4) == x[-4]..x[-3]..x[-2]..x[-1]</code></pre>
<p>The string <i>metatable</i> however is something else, and to access that you must use getmetatable("").
<br />The difference here is related to <a href="/page/Metamethods" title="Metamethods">Metamethods</a>, such as using (+) instead of (..) to concatenate strings.
</p>
<pre><code data-language="lua">print(&quot;Kittens&quot; + &quot; And &quot; + &quot;Puppies&quot;)

function getmetatable(&quot;&quot;).__add(str,x) return str..x end // This will work
function string.__add(str,x) return str..x end // But this will not.</code></pre>
<p>See <a href="/page/Meta_Tables" title="Meta Tables">Meta Tables</a> and <a href="/page/Metamethods" title="Metamethods">Metamethods</a> for more information.
</p>

<td>WARNING

</p>
