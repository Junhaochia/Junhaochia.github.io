<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>This is a direct binding to the function “engine-&gt;LightStyle”. This function allows you to change the default light style of the map - so you can make lighting lighter or darker. You’ll need to call <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/RedownloadAllLightmaps" title="render/RedownloadAllLightmaps">RedownloadAllLightmaps</a> clientside to refresh the lightmaps to this new color.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Calling this function with arguments 0 and "a" will cause dynamic lights such as those produced by the Light tool to stop working.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/3626">#3626</a></b>
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> lightstyle</span>
</p>
<div style="margin-left: 32px;">
The lightstyle to edit. 0 to 63. If you want to edit map lighting, you want to set this to 0.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> pattern</span>
</p>
<div style="margin-left: 32px;">
The pattern to change the lightstyle to. "a" is the darkest, "z" is the brightest. You can use stuff like "abcxyz" to make flashing patterns. The normal brightness for a map is "m". Values over "z" are allowed, "~" for instance.</div>
</div>
