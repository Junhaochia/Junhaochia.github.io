<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Adds surface properties to the game's physics environment.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>The game has a limit of 128 surface properties - this includes properties loaded automatically from <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod/blob/master/garrysmod/scripts/surfaceproperties.txt">surfaceproperties.txt</a>. Due to this, there's only a small amount of open slots that can be registered with GMod's provided surfaceproperties.txt.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/2604">#2604</a></b>
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> properties</span>
</p>
<div style="margin-left: 32px;">
The properties to add. Each one should include "base" or the game will crash due to some values being missing.</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Adds the "" surface property from TF2.
</p>
<pre><code data-language="lua">physenv.AddSurfaceData([[&quot;scout_baseball&quot;
{
	&quot;base&quot;		&quot;rubber&quot;

	&quot;bulletimpact&quot;	&quot;Weapon_Baseball.HitWorld&quot;
	&quot;scraperough&quot;	&quot;Grenade.ScrapeRough&quot;
	&quot;scrapesmooth&quot;	&quot;Grenade.ScrapeSmooth&quot;
	&quot;impacthard&quot;	&quot;Weapon_Baseball.HitWorld&quot;
	&quot;impactsoft&quot;	&quot;Weapon_Baseball.HitWorld&quot;
	&quot;rolling&quot;	&quot;Grenade.Roll&quot;
}]])</code></pre>
