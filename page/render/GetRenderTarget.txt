<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns the currently active render target.
</p><p>Instead of saving the current render target using this function and restoring to it later, it is generally better practice to use <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/PushRenderTarget" title="render/PushRenderTarget">PushRenderTarget</a> and <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/PopRenderTarget" title="render/PopRenderTarget">PopRenderTarget</a>.
</p>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Render something to a different render target, then restore the old render target
</p>
<pre><code data-language="lua">local w, h = ScrW(), ScrH()
local customRt = GetRenderTarget( &quot;some_unique_render_target_nameeeee&quot;, w, h, true )

render.PushRenderTarget( customRt )
    render.Clear( 0, 0, 255, 255, true ) -- fill the background with blue!

    -- draw all props on the blue background!
    for key, prop in pairs(ents.FindByClass( &quot;prop_physics&quot; )) do
        prop:DrawModel()
    end

    -- save the picture to the garrysmod/data folder.  ~format=&quot;jpg&quot; will not work.
    local data = render.Capture({ format = &quot;jpeg&quot;, quality = 70, x = 0, y = 0, h = h, w = w })	
    local pictureFile = file.Open( &quot;RenderTargetsAreAwesome.jpg&quot;, &quot;wb&quot;, &quot;DATA&quot; )	
    pictureFile:Write( data )
    pictureFile:Close()
render.PopRenderTarget()</code></pre>
