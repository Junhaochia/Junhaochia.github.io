<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Draws a sphere in 3D space. The material previously set with <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/SetMaterial" title="render/SetMaterial">SetMaterial</a> will be applied the sphere's surface.
</p><p>See also <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/DrawWireframeSphere" title="render/DrawWireframeSphere">DrawWireframeSphere</a> for a wireframe equivalent.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This is a rendering function that requires a 3D rendering context.<br />This means that it will only work in <a href="/page/Category_colon_3D_Rendering_Hooks" title="Category:3D Rendering Hooks">hooks with a 3D rendering context</a>.
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_Vector" title="Category:Vector">Vector</a> position</span>
</p>
<div style="margin-left: 32px;">
Position of the sphere.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> radius</span>
</p>
<div style="margin-left: 32px;">
Radius of the sphere. Negative radius will make the sphere render inwards rather than outwards.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_3">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> longitudeSteps</span>
</p>
<div style="margin-left: 32px;">
The number of longitude steps. This controls the quality of the sphere. Higher quality will lower performance significantly. 50 is a good number to start with.</div>
</div>

</p>
<div style="margin-left: 32px;">
The number of latitude steps. This controls the quality of the sphere. Higher quality will lower performance significantly. 50 is a good number to start with.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_5">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_table" title="Category:table">table</a> color=<a href="/page/Global/Color" title="Global/Color">Color</a>( 255, 255, 255 )</span>
</p>
<div style="margin-left: 32px;">
The color of the sphere. Uses the <a href="/page/Structures/Color" title="Structures/Color">Color structure</a>.</div>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>This will draw a blue, half-translucent sphere (force field) at the position local player is looking.
</p>
<pre><code data-language="lua">hook.Add( &quot;PostDrawTranslucentRenderables&quot;, &quot;test&quot;, function()

	--[[
		when you draw a sphere, you have to specify what material the sphere is
		going to have before rendering it, render.SetColorMaterial()
		just sets it to a white material so we can recolor it easily.
	]]
	render.SetColorMaterial()

	-- The position to render the sphere at, in this case, the looking position of the local player
	local pos = LocalPlayer():GetEyeTrace().HitPos

	-- Draw the sphere!
	render.DrawSphere( pos, 50, 30, 30, Color( 0, 175, 175, 100 ) )

end )</code></pre>
