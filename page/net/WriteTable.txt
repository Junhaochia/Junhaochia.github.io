<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Appends a table to the current net message. Adds 16 extra bits per key/value pair so you're better off writing each individual key/value as the exact type if possible.
</p>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>All net messages have a 64kb buffer. This function will not check or error when that buffer is overflown. You might want to consider using <a href="/page/Category_colon_util" title="Category:util">util</a>.<a href="/page/util/TableToJSON" title="util/TableToJSON">TableToJSON</a> and <a href="/page/Category_colon_util" title="Category:util">util</a>.<a href="/page/util/Compress" title="util/Compress">Compress</a> and send the resulting string in 60kb chunks, doing the opposite on the receiving end.
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_table" title="Category:table">table</a> table</span>
</p>
<div style="margin-left: 32px;">
<p>The table to be sent.
</p>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>If the table contains a nil key the table may not be read correctly.
</p><p>Not all objects can be sent over the network. Things like functions, <a href="/page/Category_colon_IMaterial" title="Category:IMaterial">IMaterials</a>, etc will cause errors when reading the table from a net message.
</p>
</td></tr></table></div>
