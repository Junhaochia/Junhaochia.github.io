<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Loads a .gwen file (created by GWEN Designer) and calls <a href="/page/Category_colon_Panel" title="Category:Panel">Panel</a>:<a href="/page/Panel/LoadGWENString" title="Panel/LoadGWENString">LoadGWENString</a> with the contents of the loaded file.
</p><p>Used to load panel controls from a file.
</p>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> filename</span>
</p>
<div style="margin-left: 32px;">
The file to open. The path is relative to garrysmod/garrysmod/.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> path="GAME"</span>
</p>
<div style="margin-left: 32px;">
<p>The path used to look up the file.
</p>
<ul><li> "GAME" Structured like base folder (garrysmod/), searches all the mounted content (main folder, addons, mounted games etc)
</li><li> "LUA" or "lsv" - All Lua folders (lua/) including gamesmodes and addons
</li><li> "DATA" Data folder (garrysmod/data)
</li><li> "MOD" Strictly the game folder (garrysmod/), ignores mounting.</div>
</li></ul>
</div>
