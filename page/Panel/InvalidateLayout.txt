<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Causes the panel to re-layout in the next frame. During the layout process <a href="/page/Category_colon_PANEL_Hooks" title="Category:PANEL Hooks">PANEL</a>:<a href="/page/PANEL/PerformLayout" title="PANEL/PerformLayout">PerformLayout</a> will be called on the target panel.
</p><p>You should avoid calling this function every frame.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Using this on a panel after clicking on a docked element will cause docked elements to reorient themselves incorrectly. This can be fixed by assigning a unique <a href="/page/Category_colon_Panel" title="Category:Panel">Panel</a>.<a href="/page/Panel/SetZPos" title="Panel/SetZPos">SetZPos</a> to each docked element.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/2574">#2574</a></b>
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_boolean" title="Category:boolean">boolean</a> layoutNow=false</span>
</p>
<div style="margin-left: 32px;">
If true the panel will re-layout instantly and not wait for the next frame.</div>
</div>
