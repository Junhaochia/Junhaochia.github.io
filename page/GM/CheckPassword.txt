<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Called when a non local player connects to allow the Lua system to check the password.
</p><p>The default behaviour in the base gamemodes emulates what would normally happen. If sv_password is set and its value matches the password passed in by the client - then they are allowed to join. If it isn't set it lets them in too.
</p>

</p>

The 64bit Steam ID of the joining player, use <a href="/page/Category_colon_util" title="Category:util">util</a>.<a href="/page/util/SteamIDFrom64" title="util/SteamIDFrom64">SteamIDFrom64</a> to convert it to a "STEAM_0:" one.</div>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>A user access whitelist to the server
</p><p>Available pre-defined messages can be found in <i>../sourceengine/resource/gameui_english.txt</i> files.
</p><p>Suggested messages are <i>#GameUI_ConnectionFailed</i> and <i>#GameUI_ServerRejectLANRestrict</i>
</p>
<pre><code data-language="lua">local allowed = {
	[&quot;76561198012345678&quot;] = true, -- Me
	[&quot;76561198123456789&quot;] = true, -- Friend #1
	[&quot;76561198234567890&quot;] = true, -- Friend #2
}

hook.Add( &quot;CheckPassword&quot;, &quot;access_whitelist&quot;, function( steamID64 )
	if not allowed[steamID64] then
		return false, &quot;#GameUI_ServerRejectLANRestrict&quot;
	end
end )</code></pre>
