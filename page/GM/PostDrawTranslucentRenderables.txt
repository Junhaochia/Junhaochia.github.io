<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Called after all translucent entities are drawn.
</p><p>See also <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/PostDrawOpaqueRenderables" title="GM/PostDrawOpaqueRenderables">PostDrawOpaqueRenderables</a> and <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/PreDrawTranslucentRenderables" title="GM/PreDrawTranslucentRenderables">PreDrawTranslucentRenderables</a>.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This is a rendering hook with a 3D rendering context.<br />This means that the only rendering functions will work in it are <a href="/page/Category_colon_3D_Rendering_Functions" title="Category:3D Rendering Functions">functions with a 3D rendering context</a>.
</p>
</td></tr></table>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Draws a solid black sphere at where the player is looking at, but not when the skybox is being drawn.
</p><p>You can see why this is needed if you disable the skybox check and look into the sky on gm_flatgrass (or any other map where the 3d skybox is below the map) and you will notice 2 spheres and not 1.
</p>
<pre><code data-language="lua">hook.Add( &quot;PostDrawTranslucentRenderables&quot;, &quot;test&quot;, function( bDepth, bSkybox )

	-- If we are drawing in the skybox, bail
	if ( bSkybox ) then return end

	-- Set the draw material to solid white
	render.SetColorMaterial()

	-- The position to render the sphere at, in this case, the looking position of the local player
	local pos = LocalPlayer():GetEyeTrace().HitPos

	-- Draw the sphere!
	render.DrawSphere( pos, 500, 30, 30, Color( 0, 0, 0 ) )

end )</code></pre>
