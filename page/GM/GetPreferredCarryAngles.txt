<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Called to determine preferred carry angles for the entity. It works for both, +use pickup and gravity gun pickup.
</p>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>Due to nature of the gravity gun coding in multiplayer, this hook <b>MAY</b> seem to not work ( but rest assured it does ), due to clientside prediction not knowing the carry angles. The +use pickup doesn't present this issue as it doesn't predict the player carrying the object clientside ( as you may notice by the prop lagging behind in multiplayer )
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This hook can <b>not</b> override preferred carry angles of props such as the sawblade and the harpoon.
</p>
</td></tr></table>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Makes all pickupable entities default to Angle( 0, 0, 0 ) relatively to players aim direction.
</p>
<pre><code data-language="lua">hook.Add( &quot;GetPreferredCarryAngles&quot;, &quot;MyPreferredCarryAngles&quot;, function( ent )
	return Angle( 0, 0, 0 )
end )</code></pre>
