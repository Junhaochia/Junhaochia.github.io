<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Called to determine if the <a href="/page/Global/LocalPlayer" title="Global/LocalPlayer">LocalPlayer</a> should be drawn.
</p><p>Due to an <a rel="nofollow" class="external text" href="https://garry.tv/2012/10/30/optimising-gmod/">optimization</a>, this hook is only called once per frame (<a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/3092">github issue</a>). This is problematic if you need to have the player drawn only in certain contexts, such as within <a href="/page/Category_colon_render" title="Category:render">render</a>.<a href="/page/render/RenderView" title="render/RenderView">RenderView</a> or based on the render target. As a workaround, you can call <kbd><a href="/page/Category_colon_cam" title="Category:cam">cam</a>.<a href="/page/cam/Start3D" title="cam/Start3D">Start3D</a>()&#160;<a href="/page/Category_colon_cam" title="Category:cam">cam</a>.<a href="/page/cam/End3D" title="cam/End3D">End3D</a>()</kbd> within this hook to force the engine to call it every time.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>If you're using this hook to draw a player for a <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/CalcView" title="GM/CalcView">CalcView</a> hook, then you may want to consider using the drawviewer variable you can use in your <a href="/page/Structures/CamData" title="Structures/CamData">CamData structure</a> table instead.
</p>
</td></tr></table>
