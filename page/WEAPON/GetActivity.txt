<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns the sequence enumeration number that the weapon is playing.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>This can return inconsistent results between the server and client.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/2543">#2543</a></b>
</p>
</td></tr></table>
<h1 class="first_ret"> <span class="mw-headline" id="Returns">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a></span>
</p>
<div style="margin-left: 32px;">
Current activity, see <a href="/page/Enums/ACT" title="Enums/ACT">ACT_ Enums</a>. Returns 0 if the weapon doesn't have active sequence.</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>This will return the ACT_ENUM that is currently active for the weapon.
</p>
<pre><code data-language="lua">local wep = Entity(1):GetActiveWeapon()
 
if ( IsValid( wep ) ) then // Makes sure that wep exists
 
    print( wep:GetActivity() ) // Prints the sequence number
 
end</code></pre>
