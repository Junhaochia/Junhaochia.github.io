<h1> <span class="mw-headline" id="Introduction">Introduction</span></h1>
<p>A lua file is called a script, it's plain text. To create and edit these scripts you need a plain text editor such as notepad.
</p><p>You can use any text editor, but to make your life easier, we recommend that you use Notepad++. A plugin you may want to use with Notepad++ would definitely be the Gmod Lua highlighter: <a rel="nofollow" class="external free" href="https://github.com/kylefleming/npp-gmod-lua">https://github.com/kylefleming/npp-gmod-lua</a>
</p><p>For the sake of simplicity we'll refer to both as "notepad".
</p><p><i><b>Warning: Do not use Wordpad.</b> It is a formatted text editor, not a plain text editor. It will put useless tags in around your code that will cause it to fail when loading. As a rule of thumb, if you can have text that uses two different fonts in one file, then it is not a plain text editor.</i>
</p>
<h1> <span class="mw-headline" id="Creating_the_script">Creating the script</span></h1>
<p>For our first script, we're not going to do anything too complex. We'll learn how to send a message to the console.
</p><p>Type the following code into your chosen editor&#160;:
</p>
<pre><code data-language="lua">print( &quot;Hello world&quot; )</code></pre>
<p>You're done! Wasn't that easy? It should have been.
</p>
<h2> <span class="mw-headline" id="Explanation">Explanation</span></h2>
<p><b>print</b> is a function. A function is a command that does something when you call it. Many functions can take arguments, which is data you give the function to change exactly what it does. In this case, print() takes only one argument, which is a string (a series of letters, numbers, spaces, and so on), and when print() is called, it puts that string into the console in Garry's Mod.
</p>
<h1> <span class="mw-headline" id="Saving_the_script">Saving the script</span></h1>
<p>You're now ready to create the actual Lua script file. To find your lua folder follow the following path - this may be slightly different on your computer, but will look something like this:
</p><p><code>C:\Program Files\Steam\steamapps\common\garrysmod\garrysmod\lua\</code>
</p><p>In the filename box, type helloworld.lua (Note that you must specify .lua), and in the Save As type box, select All Files. Now, just press enter (or press the save button) to save your script.
</p>
<h1> <span class="mw-headline" id="Running_the_script">Running the script</span></h1>
<p>To run any of your scripts you need to be playing a map. Now, open the console and type the following&#160;:
</p><p><code>lua_openscript helloworld.lua</code>
</p><p>Press enter. If you did everything correctly up to this point you should see a message in the console&#160;:
</p><p><code>Hello World!</code>
</p>
<h1> <span class="mw-headline" id="Auto_running_the_script">Auto running the script</span></h1>
<p>To make the script run automatically when the server starts, you simply put it into one of those folders:
</p>
<ul><li> For shared scripts ( Loaded on both client and server )
</li></ul>
<p><code>C:\Program Files\Steam\steamapps\common\garrysmod\garrysmod\lua\autorun</code>
</p>
<ul><li> For client-only scripts ( Loaded only on client )
</li></ul>
<p><code>C:\Program Files\Steam\steamapps\common\garrysmod\garrysmod\lua\autorun\client</code>
</p>
<ul><li> For server-only scripts ( Loaded only on server )
</li></ul>
<p><code>C:\Program Files\Steam\steamapps\common\garrysmod\garrysmod\lua\autorun\server</code>
</p><p>For this specific tutorial any of those paths will work just fine.
</p><p>Note, that all client and shared files must be put for download using <a href="/page/Global/AddCSLuaFile" title="Global/AddCSLuaFile">AddCSLuaFile</a> for them to work in multiplayer.
</p>
