<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Adds the specified and all related files to the files the client should download.
</p><p>For convenience, this function will automatically add any other files that are related to the selected one, and throw an error if it can't find them. For example, a .vmt file will automatically add the .vtf with the same name, and a .mdl file will automatically add all .vvd, .ani, .dx80.vtx, .dx90.vtx, .sw.vtx, .phy and .jpg files with the same name, with a separate error for each missing file. If you do not want it to do this, use <a href="/page/resource/AddSingleFile" title="resource/AddSingleFile">resource.AddSingleFile</a>.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>There's a 8192 downloadable file limit. If you need more, consider using Workshop addons - <a href="/page/Category_colon_resource" title="Category:resource">resource</a>.<a href="/page/resource/AddWorkshop" title="resource/AddWorkshop">AddWorkshop</a>. You should also consider the fact that you have way too many downloads. This limit is shared among all resource.Add functions.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Running this function using a path relative to the main garrysmod/ folder will not work for files in addons and gamemodes. Instead, the files must be added relative to their respective content folders. For example, a sound file from an addon would use the path "sound/[FILENAME_AND_EXTENSION]," despite the file being located in "addons/[ADDON_NAME]/sound/[FILENAME_AND_EXTENSION]"
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> path</span>
</p>
<div style="margin-left: 32px;">
Virtual path of the file to be added, relative to garrysmod/. Do not add .bz2 to the filepath. Do not put gamemodes/&lt;gamemodename&gt;/content/ or addons/&lt;addonname&gt;/ into the path.</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Example of usage.
</p>
<pre><code data-language="lua">resource.AddFile( &quot;materials/my/material.vmt&quot; ) -- Automatically adds materials/my/material.vtf
resource.AddFile( &quot;models/my/model.mdl&quot; ) -- Automatically adds models/my/model.vtx and the rest
resource.AddFile( &quot;sound/my/sound.wav&quot; ) -- Be careful, there's no S in the sound.</code></pre>
