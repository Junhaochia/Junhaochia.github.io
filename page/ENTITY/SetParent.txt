<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Sets the parent of this entity, making it move with its parent.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This does not work on <a href="/page/game/GetWorld" title="game/GetWorld">the world</a>.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>This can cause undefined physics behaviour when used on entities that don't support parenting. See the <a rel="nofollow" class="external text" href="https://developer.valvesoftware.com/wiki/Entity_Hierarchy_(parenting)">Valve developer wiki</a> for more information.
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a> parent=NULL</span>
</p>
<div style="margin-left: 32px;">
The entity to parent to. Setting this to nil will clear the parent.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> attachmentId=-1</span>
</p>
<div style="margin-left: 32px;">
<p>The attachment id to use when parenting, defaults to -1 or whatever the parent had set previously.<br />
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>You must call <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetMoveType" title="Entity/SetMoveType">SetMoveType</a>( MOVETYPE_NONE ) on the child for this argument to have any effect!
</p>
</td></tr></table></div>
