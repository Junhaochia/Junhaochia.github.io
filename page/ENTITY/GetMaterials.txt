<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns all materials of the entity's model.
</p><p>This function is unaffected by <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetSubMaterial" title="Entity/SetSubMaterial">SetSubMaterial</a> as it returns the original materials.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>The server's values take priority on the client.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/3362">#3362</a></b>
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>The table returned by this function will not contain materials if they are missing from the disk/repository. This means that if you are attempting to find the ID of a material to replace with <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetSubMaterial" title="Entity/SetSubMaterial">SetSubMaterial</a> and there are missing materials on the model, all subsequent materials will be offset in the table, meaning that the ID you are trying to get will be incorrect.
</p>
</td></tr></table>
