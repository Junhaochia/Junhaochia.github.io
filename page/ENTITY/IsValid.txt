<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns whether the entity is a valid entity or not.
</p><p>An entity is valid if:
</p>
<ul><li> It is not a <a href="/page/Global_Variables" title="Global Variables">NULL</a> entity
</li><li> It is not the worldspawn entity (<a href="/page/Category_colon_game" title="Category:game">game</a>.<a href="/page/game/GetWorld" title="game/GetWorld">GetWorld</a>)
</li></ul>
<p>Instead of calling this method directly, it's a good idea to call the global <a href="/page/Global/IsValid" title="Global/IsValid">IsValid</a> instead.<br />It will check whether the given variable contains an object (an Entity) or nothing at all for you. See examples.
</p>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>NULL entities can still be assigned with key/value pairs, but they will be instantly negated. See example 3
</p>
</td></tr></table>
<p>This might be a cause for a lot of headache. Usually happening during networking etc., when completely valid entities suddenly become invalid on the client, but are never filtered with IsValid(). See <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/InitPostEntity" title="GM/InitPostEntity">InitPostEntity</a> for more details.
</p>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Shows how to use the global <a href="/page/Global/IsValid" title="Global/IsValid">IsValid</a> function instead of using this method directly.
</p>
<pre><code data-language="lua">if ( entity &amp;&amp; entity:IsValid() ) then
	-- Do stuff
end

-- The above can be replaced with the following for the same effect (and cleaner code)

if ( IsValid( entity ) ) then
	-- Do stuff
end</code></pre>
<p><br />
</p>
<h1 class="first_example"> <span class="mw-headline" id="Examples_2">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example_2">Example</span></h2>
<pre><code data-language="lua">print( LocalPlayer():IsValid() )</code></pre>
<p><b>Output:</b>
</p>
<div style="margin:2em;">Outputs 'true' to the console if the player is in-game.</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples_3">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example_3">Example</span></h2>
<pre><code data-language="lua">local newPlayer = net.ReadEntity() --server found a new player on the server and sent it to us after 
print( &quot;1/4&quot; )              -- it determined it was valid (newPlayer is NULL in this realm)

if not isnumber(newPlayer.ImportantGameData) then --If it doesn't have a specific field, assign a value to it
	print( &quot;2/4&quot; )
	newPlayer.ImportantGameData = 42 
end
print( &quot;3/4&quot; )

print( newPlayer.ImportantGameData * 69 ) --Attempting to do arithmetics on the new field
print( &quot;4/4&quot; )</code></pre>
<p><b>Output:</b>
</p>
<div style="margin:2em;">1/4 ... 3/4
<ul><li>A lua error telling us 'ImportantGameData' is a nil value and thus can't perform math on it
</li></ul>
<hr />
But this shouldn't be possible, because we just created a value there. No red flags are present up until this point; all code up until this will run fine. Make sure to add an IsValid() check when the seemingly impossible happens.</div>
