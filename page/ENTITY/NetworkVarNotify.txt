<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Creates a callback that will execute when the given network variable changes - that is, when the Set&lt;name&gt; function is run.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>The callback is executed <i>before</i> the value is changed, and is called even if the new and old values are the same.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This function does not exist on entities in which <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/InstallDataTable" title="Entity/InstallDataTable">InstallDataTable</a> has not been called. By default, this means this function only exists on SENTs (both serverside and clientside) and on players with a <a href="/page/Player_Classes" title="Player Classes">Player Class</a> (serverside and clientside <a href="/page/Global/LocalPlayer" title="Global/LocalPlayer">LocalPlayer</a> only!). It is therefore safest to only use this in <a href="/page/Category_colon_ENTITY_Hooks" title="Category:ENTITY Hooks">ENTITY</a>:<a href="/page/ENTITY/SetupDataTables" title="ENTITY/SetupDataTables">SetupDataTables</a>.
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>The callback will not be called clientside if the var is changed right after entity spawn.
</p><p><b>Request Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-requests/issues/324">#324</a></b>
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> name</span>
</p>
<div style="margin-left: 32px;">
Name of variable to track changes of</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_function" title="Category:function">function</a> callback</span>
</p>
<div style="margin-left: 32px;">
<p>The function to call when the variable changes. It is passed 4 arugments:
</p>
<ul><li> <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a> entity - Entity whos variable changed (This will be variable called "self" in ENT:CallBack format.)
</li><li> <a href="/page/Category_colon_string" title="Category:string">string</a> name - Name of changed variable
</li><li> <a href="/page/Category_colon_any" title="Category:any">any</a> old - Old/current variable value
</li><li> <a href="/page/Category_colon_any" title="Category:any">any</a> new - New variable value that it was set to</div>
</li></ul>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Example usage
</p>
<pre><code data-language="lua">function ENT:SetupDataTables()

	self:NetworkVar( &quot;Float&quot;, 0, &quot;Amount&quot; )
	self:NetworkVar( &quot;Vector&quot;, 1, &quot;StartPos&quot; )
	self:NetworkVar( &quot;Vector&quot;, 2, &quot;EndPos&quot; )

	if ( SERVER ) then
		self:NetworkVarNotify( &quot;EndPos&quot;, self.OnVarChanged )
	end

end

function ENT:OnVarChanged( name, old, new )
	print( name, old, new )
end</code></pre>
<p><b>Output:</b>
</p>
<div style="margin:2em;">Prints variable name, old value and new value whenever SetEndPos function is called</div>
