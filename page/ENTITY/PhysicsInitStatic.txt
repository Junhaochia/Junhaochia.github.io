<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Initializes a static physics object of the entity using its <a href="/page/Entity/GetModel" title="Entity/GetModel">current model</a>. If successful, the previous physics object is removed.
</p><p>This is what used by entities such as func_breakable, prop_dynamic, item_suitcharger, prop_thumper and npc_rollermine while it is in its "buried" state in the Half-Life 2 Campaign.
</p><p>If the entity's current model has no physics mesh associated to it, no physics object will be created.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This function will automatically call <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetSolid" title="Entity/SetSolid">SetSolid</a>( solidType ).
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Clientside physics objects are broken and do not move properly in some cases. Physics objects should only created on the server or you will experience incorrect physgun beam position, prediction issues, and other unexpected behavior.
</p>
</td></tr></table>
