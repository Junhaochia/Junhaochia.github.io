<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>When called on a constraint entity, sets the two physics objects to be constrained.
</p><p>Usage is not recommended as the Constraint library provides easier ways to deal with constraints.
</p>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_PhysObj" title="Category:PhysObj">PhysObj</a> Phys1</span>
</p>
<div style="margin-left: 32px;">
The first physics object to be constrained.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_PhysObj" title="Category:PhysObj">PhysObj</a> Phys2</span>
</p>
<div style="margin-left: 32px;">
The second physics object to be constrained.</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>From constraint.lua
</p>
<pre><code data-language="lua">Constraint = ents.Create(&quot;phys_lengthconstraint&quot;)
	Constraint:SetPos( WPos1 )
	Constraint:SetKeyValue( &quot;attachpoint&quot;, tostring(WPos2) )
	Constraint:SetKeyValue( &quot;minlength&quot;, &quot;0.0&quot; )
	Constraint:SetKeyValue( &quot;length&quot;, length + addlength )
	if ( forcelimit ) then Constraint:SetKeyValue( &quot;forcelimit&quot;, forcelimit ) end
	if ( rigid ) then Constraint:SetKeyValue( &quot;spawnflags&quot;, 2 ) end
	Constraint:SetPhysConstraintObjects( Phys1, Phys2 )
Constraint:Spawn()
Constraint:Activate()</code></pre>
