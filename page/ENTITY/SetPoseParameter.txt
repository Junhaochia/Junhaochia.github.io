<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Sets the specified pose parameter to the specified value.
</p><p>You should call <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/InvalidateBoneCache" title="Entity/InvalidateBoneCache">InvalidateBoneCache</a> after calling this function.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Avoid calling this in draw hooks, especially when animating things, as it might cause visual artifacts.
</p>
</td></tr></table>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Copies pose parameters from one entity to another. Since <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>.<a href="/page/Entity/GetPoseParameter" title="Entity/GetPoseParameter">GetPoseParameter</a> returns pose parameter values 0-1 on the client, they have to be remapped to the range returned by <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>.<a href="/page/Entity/GetPoseParameterRange" title="Entity/GetPoseParameterRange">GetPoseParameterRange</a> before being set on the target entity.
</p>
<pre><code data-language="lua">local function CopyPoseParams(pEntityFrom, pEntityTo)
	if (SERVER) then
		for i = 0, pEntityFrom:GetNumPoseParameters() - 1 do
			local sPose = pEntityFrom:GetPoseParameterName(i)
			pEntityTo:SetPoseParameter(sPose, pEntityFrom:GetPoseParameter(sPose))
		end
	else
		for i = 0, pEntityFrom:GetNumPoseParameters() - 1 do
			local flMin, flMax = pEntityFrom:GetPoseParameterRange(i)
			local sPose = pEntityFrom:GetPoseParameterName(i)
			pEntityTo:SetPoseParameter(sPose, math.Remap(pEntityFrom:GetPoseParameter(sPose), 0, 1, flMin, flMax))
		end
	end
end</code></pre>
