<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Makes the physics object of the entity a AABB.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>If the volume of the resulting box is 0 (the mins and maxs are the same), the mins and maxs will be changed to <a href="/page/Global/Vector" title="Global/Vector">Vector</a>( -1, -1, -1 ) and <a href="/page/Global/Vector" title="Global/Vector">Vector</a>( 1, 1, 1 ), respectively.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This function will automatically destroy any previous physics objects if successful and call <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetSolid" title="Entity/SetSolid">SetSolid</a>( SOLID_BBOX ), <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetMoveType" title="Entity/SetMoveType">SetMoveType</a>( MOVETYPE_VPHYSICS ), and <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetCollisionBounds" title="Entity/SetCollisionBounds">SetCollisionBounds</a>( mins, maxs ).
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Clientside physics objects are broken and do not move properly in some cases. Physics objects should only created on the server or you will experience incorrect physgun beam position, prediction issues, and other unexpected behavior.
</p>
</td></tr></table>

</div>
