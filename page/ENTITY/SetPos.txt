<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Moves the entity to the specified position.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>If the new position doesn't take effect right away, you can use <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/SetupBones" title="Entity/SetupBones">SetupBones</a> to force it to do so. This issue is especially common when trying to render the same entity twice or more in a single frame at different positions.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>Entities with <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/GetSolid" title="Entity/GetSolid">GetSolid</a> of SOLID_BBOX will have their angles reset!
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>This will fail inside of predicted functions called during player movement processing. This includes <a href="/page/Category_colon_WEAPON_Hooks" title="Category:WEAPON Hooks">WEAPON</a>:<a href="/page/WEAPON/PrimaryAttack" title="WEAPON/PrimaryAttack">PrimaryAttack</a> and <a href="/page/Category_colon_WEAPON_Hooks" title="Category:WEAPON Hooks">WEAPON</a>:<a href="/page/WEAPON/Think" title="WEAPON/Think">Think</a>.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/2447">#2447</a></b>
</p>
</td></tr></table>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Sets the player's position to (0, 0, 0)
</p>
<pre><code data-language="lua">Entity( 1 ):SetPos( Vector( 0, 0, 0 ) )</code></pre>
<p><b>Output:</b>
</p>
<div style="margin:2em;">The player is now located at Vector(0, 0, 0)</div>
