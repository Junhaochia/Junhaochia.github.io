<h1> <span class="mw-headline" id="Introduction"> Introduction </span></h1>
<p>As a developer you may be familiar with <a href="/page/Category_colon_PANEL_Hooks" title="Category:PANEL Hooks">PANEL</a>:<a href="/page/PANEL/Paint" title="PANEL/Paint">Paint</a> and <a href="/page/Category_colon_PANEL_Hooks" title="Category:PANEL Hooks">PANEL</a>:<a href="/page/PANEL/PaintOver" title="PANEL/PaintOver">PaintOver</a>. They are functions to draw your Derma objects. But one day you wanted to make your own Derma design. You may think that redrawing is too complicated: <a href="/page/Category_colon_surface" title="Category:surface">surface library</a>, <a href="/page/Category_colon_draw" title="Category:draw">draw library</a>, <a href="/page/Category_colon_render" title="Category:render">render library</a>. So many functions, aren't they? Is there a way to make it easier and fancy? Yes, this page will show you how to create your own Derma Skin. Exactly same method uses default Derma and it's defined in Lua.
</p>
<h1> <span class="mw-headline" id="How_Derma_skins_work.3F"> How Derma skins work? </span></h1>
<p>First thing to know, they're located in <i>"garrysmod/lua/skins/"</i>. Default Derma skin is in "default.lua".<br />
It contains <a rel="nofollow" class="external text" href="https://en.wikipedia.org/wiki/UV_mapping">UV texture mapping</a>, <a href="/page/Global/Color" title="Global/Color">Color</a> definitions and <a rel="nofollow" class="external text" href="/page/derma/SkinHook">Derma Hooks</a>.<br />
UV mapping is possible through <a href="/page/Category_colon_GWEN" title="Category:GWEN">GWEN library</a>. Some GWEN functions require a <a href="/page/Category_colon_IMaterial" title="Category:IMaterial">IMaterial</a> in last argument, but it can be empty: GWEN will use <b>SKIN.GwenTexture</b> variable instead. Default skin texture can be found in <b>Garry's Mod .vpk</b> file in <i>"materials/gwenskin/gmoddefault.png"</i>.<br />
Inside file there're bunch of Derma Hooks. They're used in conjunction with <a href="/page/Category_colon_derma" title="Category:derma">derma</a>.<a href="/page/derma/SkinHook" title="derma/SkinHook">SkinHook</a> (used in <a href="/page/Global/Derma_Hook" title="Global/Derma Hook">Derma_Hook</a>).
</p>
<h2> <span class="mw-headline" id="Example">Example</span></h2>
<p>This is in <b>"default.lua"</b>.
</p>
<div style="background-color:#eee; border-radius:10px; padding: 10px 1px 10px 10px; font-size: 12px; font-family:Consolas,courier new;">
<p><span class="client_m" title="Client and Menu">&#160;</span>
</p>
<pre><code data-language="lua">
function SKIN:PaintFrame( panel, w, h )

	if ( panel.m_bPaintShadow ) then -- This is DFrame shadow

		DisableClipping( true )
		SKIN.tex.Shadow( -4, -4, w+10, h+10 ) -- This is drawing function, that contains a piece of our texture
		DisableClipping( false )

	end

	if ( panel:HasHierarchicalFocus() ) then -- Checks if DFrame is in focus and decide how to paint it

		self.tex.Window.Normal( 0, 0, w, h ) -- This is drawing function, that contains a piece of our texture

	else

		self.tex.Window.Inactive( 0, 0, w, h ) -- This is drawing function, that contains a piece of our texture

	end

end
</code></pre></div>
<p>And this is in <i>"lua/vgui/<b>dframe.lua</b>"</i> where <a href="/page/Category_colon_DFrame" title="Category:DFrame">DFrame</a> is defined.
</p>
<div style="background-color:#eee; border-radius:10px; padding: 10px 1px 10px 10px; font-size: 12px; font-family:Consolas,courier new;">
<p><span class="client_m" title="Client and Menu">&#160;</span>
</p>
<pre><code data-language="lua">
function PANEL:Paint( w, h )

    -- ...

	derma.SkinHook( &quot;Paint&quot;, &quot;Frame&quot;, self, w, h ) -- This creates hook to derma library, derma library is connected to our skin

    -- ...

end
</code></pre></div>
<hr />
<p>Last thing that skin file does is adding skin to Derma skins list. This action is done by <a href="/page/Category_colon_derma" title="Category:derma">derma</a>.<a href="/page/derma/DefineSkin" title="derma/DefineSkin">DefineSkin</a>.
</p>
<h1> <span class="mw-headline" id="Preparing_to_create_Derma_skin"> Preparing to create Derma skin </span></h1>
<ol><li> Get <a rel="nofollow" class="external text" href="https://www.adobe.com/products/photoshop.html">Adobe Photoshop</a>
</li><li> Extract <i>"materials/gwenskin/gmoddefault.psd"</i> (You can do this with <a rel="nofollow" class="external text" href="http://nemesis.thewavelength.net/index.php?p=26">GCFScape</a>)
</li><li> Copy code (or file) from <i>"lua/skins/<b>default.lua</b>"</i>
</li><li> Create lua file in <i>"lua/skins/"</i>. For this tutorial we will name it <b>"myskin.lua"</b>.
</li><li> Create lua file in <i>"lua/autorun/"</i>. You can name it how you want it, just make it unique, for example <b>"myskin_include.lua"</b>.
</li></ol>

<h1> <span class="mw-headline" id="Tester_snippet">Tester snippet</span></h1>
<p>To test out your Derma skin you can use this snippet.
</p>
<div style="background-color:#eee; border-radius:10px; padding: 10px 1px 10px 10px; font-size: 12px; font-family:Consolas,courier new;">
<p><span class="client" title="Client">&#160;</span>
</p>
<pre><code data-language="lua">
concommand.Add( &quot;derma_setskin&quot;, function(_,_,args)
	if GetConVar(&quot;sv_allowcslua&quot;):GetInt() == 0 then return end
	for k,v in pairs(vgui.GetWorldPanel():GetChildren()) do
		v:SetSkin(args[1])
	end
end,nil,&quot;Sets skin for all Derma objects.&quot;)
local function deepskin(children,skin)
	for k,v in pairs(children) do
		v:SetSkin(skin)
		if #v:GetChildren()&#160;!= 0 then deepskin(v:GetChildren(),skin) end
	end
end
concommand.Add( &quot;derma_deepsetskin&quot;, function(_,_,args)
	if GetConVar(&quot;sv_allowcslua&quot;):GetInt() == 0 then return end
	deepskin(vgui.GetWorldPanel():GetChildren(),args[1])
end,nil,&quot;Forces skin set for all Derma objects and their children.&quot;)
concommand.Add( &quot;derma_updateskin&quot;, function()
	if GetConVar(&quot;sv_allowcslua&quot;):GetInt() == 0 then return end
	for k,v in pairs(derma.GetSkinTable()) do
		if v.GwenTexture then
			local tex = v.GwenTexture:GetTexture(&quot;$basetexture&quot;)
			if tex then tex:Download() end
		end
	end
	derma.RefreshSkins()
end,nil,&quot;Updates skins for all Derma objects.&quot;)
</code></pre></div>
<p>Derma objects can't update themselves because skin that they used is <b>cached</b>.
Due to skins are being <b>cached</b>, Derma objects can't update themselves while you're editing. So we need to clear cache by typing command <b>derma_updateskin</b>. Also its material can't update itself, to fix that, it uses <a href="/page/Category_colon_ITexture" title="Category:ITexture">ITexture</a>:<a href="/page/ITexture/Download" title="ITexture/Download">Download</a>.<br />
Function <b>derma_setskin *skin name*</b> allows you to apply Derma skin on all Derma object through the console. If some panels didn't update use <b>derma_deepsetskin *skin name*</b>.<br />
For safety, you can't use these functions when <b>sv_allowcslua is 0</b>. You can remove this check if you don't forget to remove tester later.
To test your skin you can type <b>derma_controls</b> to preview.
</p>

<h1> <span class="mw-headline" id="Demo"> Demo </span></h1>
<p>For tutorial was created a Demo Addon with Windows 10 UI skin for Derma.
You can download it here: <a rel="nofollow" class="external text" href="https://steamcommunity.com/sharedfiles/filedetails/?id=1633684835">Download from Steam Workshop</a>
Addon uses this hook for painting all Derma objects:
</p>
<div style="background-color:#eee; border-radius:10px; padding: 10px 1px 10px 10px; font-size: 12px; font-family:Consolas,courier new;">
<p><span class="client" title="Client">&#160;</span>
</p>
<pre><code data-language="lua">
	hook.Add(&quot;ForceDermaSkin&quot;,&quot;Windows10SkinForce&quot;,function()
		return &quot;Windows&quot; -- This will paint all Derma objects to new skin
	end)
</code></pre>
</div>
