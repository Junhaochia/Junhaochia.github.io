<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns the public IP address and port of the current server. This will return the IP/port that you are connecting through when ran clientside.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Returns "0.0.0.0:0" in singleplayer.
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Returns "0.0.0.0:<i>port</i>" on the server when called too early, including in <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/Initialize" title="GM/Initialize">Initialize</a> and <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/InitPostEntity" title="GM/InitPostEntity">InitPostEntity</a>. This bug seems to only happen the first time a server is launched, and will return the correct value after switching maps.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/3001">#3001</a></b>
</p>
</td></tr></table>
