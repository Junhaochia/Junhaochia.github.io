<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Emits the specified sound at the specified position.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Sounds must be precached serverside manually before they can be played. util.PrecacheSound does not work for this purpose, Entity.EmitSound does the trick
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>This does not work with soundscripts. TODO: Is this a bug or intended?
</p>
</td></tr></table>

<h1 class="first_arg"> <span class="mw-headline" id="Arguments_3">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> entity</span>
</p>
<div style="margin-left: 32px;">
<p>The entity to emit the sound from. Can be an <a href="/page/Category_colon_Entity" title="Category:Entity">Entity</a>:<a href="/page/Entity/EntIndex" title="Entity/EntIndex">EntIndex</a> or one of the following:
</p>
<ul><li> 0 - Plays sound on the world (Position set to 0,0,0)
</li><li> -1 - Plays sound on the local player (on server acts as 0)
</li><li> -2 - Plays UI sound (Position set to 0,0,0, no spatial sound,on server acts as 0)</div>
</li></ul>
</div>
