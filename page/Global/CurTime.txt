<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns the uptime of the server in seconds (to at least 4 decimal places)
</p><p>This is a synchronised value and affected by various factors such as host_timescale (or <a href="/page/Category_colon_game" title="Category:game">game</a>.<a href="/page/game/GetTimeScale" title="game/GetTimeScale">GetTimeScale</a>) and the server being paused - either by sv_pausable or all players disconnecting.
</p><p>You should use this function for timing in-game events but not for real-world events.
</p><p>See also: <a href="/page/Global/RealTime" title="Global/RealTime">RealTime</a>, <a href="/page/Global/SysTime" title="Global/SysTime">SysTime</a>
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>This is internally defined as a float, and as such it will be affected by precision loss if your server uptime is more than 6 hours, which will cause jittery movement of players and props and inaccuracy of timers, it is highly encouraged to refresh or change the map when that happens (a server restart is not necessary).<br /><br />This is <b>NOT</b> easy as it sounds to fix in the engine, so please refrain from posting issues about this
</p>
</td></tr></table>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Simple delay timer.
</p>
<pre><code data-language="lua">local delay = 0
hook.Add( &quot;Think&quot;, &quot;CurTimeDelay&quot;, function()
 if CurTime() &lt; delay then return end
	print( &quot;This message will repeat every 5 seconds.&quot; )
	 delay = CurTime() + 5
 end)</code></pre>
<p><b>Output:</b>
</p>
<div style="margin:2em;">This message will repeat every 5 seconds.</div>
