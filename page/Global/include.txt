<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Executes a Lua script.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Addon files (.gma files) do not support relative parent folders (<i>..</i> notation).
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>The file you are attempting to include MUST NOT be empty or the include will fail. Files over a certain size may fail as well.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p>If the file you are including is clientside or shared, it <b>must</b> be <a href="/page/Global/AddCSLuaFile" title="Global/AddCSLuaFile">AddCSLuaFile</a>'d or this function will error saying the file doesn't exist.
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p><a href="/page/Global/pcall" title="Global/pcall">pcalling</a> this function will break autorefresh.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/1976">#1976</a></b>
</p>
</td></tr></table>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> fileName</span>
</p>
<div style="margin-left: 32px;">
<p>The name of the script to be executed. The path must be either relative to the current file, or be an absolute path (relative to and excluding the lua/ folder).
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Please make sure your file names are unique, the filesystem is shared across all addons, so a file named "lua/config.lua" in your addon may be overwritten by the same file in another addon.
</p>
</td></tr></table></div>
</div>
<h1 class="first_ret"> <span class="mw-headline" id="Returns">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_vararg" title="Category:vararg">vararg</a></span>
</p>
<div style="margin-left: 32px;">
Anything that the executed Lua script returns.</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Demonstrates correct and incorrect usage.
</p>
<pre><code data-language="lua">-- Correct usage:
-- Will look for &quot;lua/myLuaFolder/myLuaFile.lua&quot; in all addons and then the base game lua/ folder
include( &quot;myLuaFolder/myLuaFile.lua&quot; )

-- This is incorrect, and will NOT work
include(               &quot;lua/myLuaFolder/myLuaFile.lua&quot; )
include(         &quot;addons/lua/myLuaFolder/myLuaFile.lua&quot; )
include( &quot;addons/MyAddon/lua/myLuaFolder/myLuaFile.lua&quot; )
include(       &quot;MyAddon/lua/myLuaFolder/myLuaFile.lua&quot; )</code></pre>
