<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Creates a non physical entity that only exists on the client. See also <a href="/page/Category_colon_ents" title="Category:ents">ents</a>.<a href="/page/ents/CreateClientProp" title="ents/CreateClientProp">CreateClientProp</a>.
</p>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Parented clientside models will become detached if the parent entity leaves the PVS.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/861">#861</a></b>
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Clientside entities are not garbage-collected, thus you must store a reference to the object and call <a href="/page/Category_colon_CSEnt" title="Category:CSEnt">CSEnt</a>:<a href="/page/CSEnt/Remove" title="CSEnt/Remove">Remove</a> manually.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/1387">#1387</a></b>
</p>
</td></tr></table>
<table class="gmodwiki_note bug_note">
<tr>
<td>BUG
</td>
<td>
<p>Clientside models will occasionally delete themselves during high server lag.
</p><p><b>Issue Tracker: <a rel="nofollow" class="external text" href="https://github.com/Facepunch/garrysmod-issues/issues/3184">#3184</a></b>
</p>
</td></tr></table>
