<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Returns if this is the first time this hook was predicted.
</p><p>This is useful for one-time logic in your SWEPs PrimaryAttack, SecondaryAttack and Reload and other <a href="/page/Category_colon_Predicted_Hooks" title="Category:Predicted Hooks">predicted hooks</a> (to prevent those hooks from being called rapidly in succession). It's also useful in a Move hook for when the client predicts movement.
</p><p>Visit <a href="/page/Prediction" title="Prediction">Prediction</a> for more information about this behavior.
</p>

<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>An override for <a href="/page/Category_colon_GM_Hooks" title="Category:GM Hooks">GM</a>:<a href="/page/GM/KeyPress" title="GM/KeyPress">KeyPress</a> in the gamemode Trouble in Terrorist Town, to work around the hook being called multiple times.
</p>
<pre><code data-language="lua">-- Note that for some reason KeyPress and KeyRelease are called multiple times
-- for the same key event in multiplayer.
function GM:KeyPress(ply, key)
   if not IsFirstTimePredicted() then return end
   if not IsValid(ply) or ply&#160;!= LocalPlayer() then return end

   if key == IN_SPEED and ply:IsActiveTraitor() then
      timer.Simple(0.05, function() RunConsoleCommand(&quot;+voicerecord&quot;) end)
   end
end</code></pre>
