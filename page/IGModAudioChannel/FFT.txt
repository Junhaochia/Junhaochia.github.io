<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Computes the <a rel="nofollow" class="external text" href="https://en.wikipedia.org/wiki/Discrete_Fourier_transform">DFT (discrete Fourier transform)</a> of the sound channel.
</p><p>The size parameter specifies the number of consecutive audio samples to use as the input to the DFT and is restricted to a power of two. A <a rel="nofollow" class="external text" href="https://en.wikipedia.org/wiki/Hann_function">Hann window</a> is applied to the input data.
</p><p>The computed DFT has the same number of frequency bins as the number of samples. Only half of this DFT is returned, since <a rel="nofollow" class="external text" href="https://en.wikipedia.org/wiki/Discrete_Fourier_transform#The_real-input_DFT">the DFT magnitudes are symmetric for real input data</a>. The magnitudes of the DFT (values from 0 to 1) are used to fill the output table, starting at index 1.
</p><p><b>Visualization protip:</b> For a size N DFT, bin k (1-indexed) corresponds to a frequency of (k - 1) / N * sampleRate.
</p><p><b>Visualization protip:</b> Sound energy is proportional to the square of the magnitudes. Adding magnitudes together makes no sense physically, but adding energies does.
</p><p><b>Visualization protip:</b> The human ear works on a logarithmic amplitude scale. You can convert to <a rel="nofollow" class="external text" href="https://en.wikipedia.org/wiki/Decibel">decibels</a> by taking 20 log<sub>10</sub> of frequency magnitudes, or 10 log<sub>10</sub> of energy. The decibel values will range from -infinity to 0.
</p>
