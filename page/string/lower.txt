<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Changes any upper-case letters in a string to lower-case letters.
</p>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> str</span>
</p>
<div style="margin-left: 32px;">
The string to convert.</div>
</div>
<h1 class="first_ret"> <span class="mw-headline" id="Returns">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a></span>
</p>
<div style="margin-left: 32px;">
The original string, with all uppercase letters replaced with their lowercase variants.</div>
</div>

<h1 class="first_example"> <span class="mw-headline" id="Examples_2">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example_2">Example</span></h2>
<p>Demonstrates a common use for string.lower - case-insensitive user input.
</p>
<pre><code data-language="lua">-- All keys in this table must be lowercase:
local products = {}
products.apple = &quot;Buy an apple!&quot;
products.banana = &quot;Buy a bunch of bananas!&quot;
products.tomato = &quot;There's also tomatoes.&quot;

-- This function is case-insensitive, meaning &quot;APPLE&quot;, &quot;apple&quot;, and &quot;APPle&quot; are all the same.
function GetProduct(userinput)
	return userinput, products[string.lower(userinput)]
end

-- Demonstration:
print(GetProduct(&quot;apple&quot;))
print(GetProduct(&quot;Apple&quot;))
print(GetProduct(&quot;APPLE&quot;))
print()
print(GetProduct(&quot;banana&quot;))
print(GetProduct(&quot;BaNaNa&quot;))</code></pre>
<p><b>Output:</b>
</p>
<pre class="plain">apple	Buy an apple!
Apple	Buy an apple!
APPLE	Buy an apple!
&#160;
banana	Buy a bunch of bananas!
BaNaNa	Buy a bunch of bananas!</code></pre>
