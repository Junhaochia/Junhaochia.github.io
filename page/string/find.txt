<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Attempts to find the specified substring in a string, uses <a href="/page/Patterns" title="Patterns">Patterns</a> by default.
</p>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> haystack</span>
</p>
<div style="margin-left: 32px;">
The string to search in.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_2">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a> needle</span>
</p>
<div style="margin-left: 32px;">
The string to find, can contain patterns if enabled.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_3">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a> startPos=1</span>
</p>
<div style="margin-left: 32px;">
The position to start the search from, can be negative start position will be relative to the end position.</div>
</div>
<h1 class="first_arg"> <span class="mw-headline" id="Arguments_4">Arguments</span></h1>
<div class="argument">
<p><span class="arg_number"></span> <span class="arg_chunk"><a href="/page/Category_colon_boolean" title="Category:boolean">boolean</a> noPatterns=false</span>
</p>
<div style="margin-left: 32px;">
Disable patterns.</div>
</div>
<h1 class="first_ret"> <span class="mw-headline" id="Returns">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a></span>
</p>
<div style="margin-left: 32px;">
Starting position of the found text, or nil if the text wasn't found</div>
</div>
<h1 class="first_ret"> <span class="mw-headline" id="Returns_2">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_number" title="Category:number">number</a></span>
</p>
<div style="margin-left: 32px;">
Ending position of found text, or nil if the text wasn't found</div>
</div>
<h1 class="first_ret"> <span class="mw-headline" id="Returns_3">Returns</span></h1>
<div class="return" style="counter-increment: returns;">
<p><span class="ret_number" style="counter-increment: returns -1;"></span> <span class="ret_chunk"><a href="/page/Category_colon_string" title="Category:string">string</a></span>
</p>
<div style="margin-left: 32px;">
Matched text for each group if patterns are enabled and used, or nil if the text wasn't found</div>
</div>
<h1 class="first_example"> <span class="mw-headline" id="Examples">Examples</span></h1>
<h2 class="examples_number"> <span class="mw-headline" id="Example">Example</span></h2>
<p>Change the word "heck" to "****" in chat messages
</p>
<pre><code data-language="lua">hook.Add( &quot;PlayerSay&quot;, &quot;NoHeckHere&quot;, function( ply, text )
	local heckStart, heckEnd = string.find( text:lower(), &quot;heck&quot; )
	if heckStart then
		local civilText = string.sub( text, 1, heckStart - 1 ) .. &quot;****&quot; .. string.sub( text, heckEnd + 1 )
		return civilText
	end
end )</code></pre>
