<h1> <span class="mw-headline" id="Description">Description</span></h1>
<p>Adds the specified string to a string table, which will cache it and network it to all clients automatically.<br />
Whenever you want to create a net message with <a href="/page/Category_colon_net" title="Category:net">net</a>.<a href="/page/net/Start" title="net/Start">Start</a>, you must add the name of that message as a networked string via this function.
</p><p>If the passed string already exists, nothing will happen and the ID of the existing item will be returned.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>Each unique network name needs to be pooled once - do not put this function call into any other functions if you're using a constant string. Preferable place for this function is in a serverside lua file, or in a shared file with the <a href="/page/Category_colon_net" title="Category:net">net</a>.<a href="/page/net/Receive" title="net/Receive">Receive</a> function.
</p>
</td></tr></table>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>The string table used for this function does not interfere with the engine string tables and has 2048 slots.
</p>
</td></tr></table>
