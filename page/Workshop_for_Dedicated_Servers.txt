<p>Dedicated servers can install addons straight from workshop collections, but it requires an addition to the startup parameter. The server will download the collection at startup.
</p>

<h1> <span class="mw-headline" id="Create_collection">Create collection</span></h1>
<p>First of all <a rel="nofollow" class="external text" href="https://steamcommunity.com/workshop/browse/?appid=4000&amp;section=collections">create a collection and find your collection ID</a>. Do note that your collection needs to be public, else, srcds won't be able to access it!
</p>
<h1> <span class="mw-headline" id="Applying_the_command-line_parameter">Applying the command-line parameter</span></h1>
<p>Append the following parameter to the server startup command-line:
</p>
<pre>+host_workshop_collection WORKSHOPID
</pre>
<p>An example could be:
</p>
<pre>+host_workshop_collection 123456789
</pre>
<h1> <span class="mw-headline" id="Example">Example</span></h1>
<p>The example below starts a dedicated server with 20 player slots. It downloads or updates all of the addons in collection <a rel="nofollow" class="external text" href="https://steamcommunity.com/sharedfiles/filedetails/?id=157384458">157384458</a>. This collection includes the Zombie Survival gamemode and a few maps. It sets the gamemode to zombiesurvival and loads one of the downloaded maps.
</p>
<pre>srcds.exe +maxplayers 20 -console +host_workshop_collection 157384458 +gamemode zombiesurvival +map zs_cleanoffice_v2
</pre>
<p>Every time you restart your server it will check that these addons are all up to date.
</p>
