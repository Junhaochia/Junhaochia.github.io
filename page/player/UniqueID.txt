<h1> <span class="mw-headline" id="Description">Description</span></h1>
<table cellspacing="0" class="gmodwiki_note warning_note">
<tr>
<td>WARNING
</td>
<td>
<p><b>This function has collisions,</b> where more than one player has the same UniqueID. It is <b>highly</b> recommended to use <a href="/page/Category_colon_Player" title="Category:Player">Player</a>:<a href="/page/Player/AccountID" title="Player/AccountID">AccountID</a>, <a href="/page/Category_colon_Player" title="Category:Player">Player</a>:<a href="/page/Player/SteamID" title="Player/SteamID">SteamID</a> or <a href="/page/Category_colon_Player" title="Category:Player">Player</a>:<a href="/page/Player/SteamID64" title="Player/SteamID64">SteamID64</a> instead, which are guaranteed to be unique to each player.
</p>
</td></tr></table>
<p>Returns a 32 bit integer that remains constant for a player across joins/leaves and across different servers. This can be used when a string is inappropriate - e.g. in a database primary key.
</p>
<table cellspacing="0" class="gmodwiki_note">
<tr>
<td>NOTE
</td>
<td>
<p>In Singleplayer, this function will always return 1.
</p>
</td></tr></table>
