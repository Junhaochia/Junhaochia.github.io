<p>Enumerations used by <a href="/page/Global/GetRenderTargetEx" title="Global/GetRenderTargetEx">GetRenderTargetEx</a>. Clientside only.<br /><br />
</p>
<div style="text-align: center;"><b>Warning!</b> Decimal values are for reference only! The actual value may be different between two Garry's Mod versions.</div>
<table class="wikitable sortable" border="1">
<tr>
<th scope="col"> Name
</th>
<th scope="col"> Decimal
</th>
<th scope="col"> Description
</th></tr>
<tr>
<td> RT_SIZE_NO_CHANGE
</td>
<td> 0
</td>
<td> Only allowed for render targets that don't want a depth buffer (because if they have a depth buffer, the render target must be less than or equal to the size of the framebuffer).
</td></tr>
<tr>
<td> RT_SIZE_DEFAULT
</td>
<td> 1
</td>
<td> Don't play with the specified width and height other than making sure it fits in the framebuffer.
</td></tr>
<tr>
<td> RT_SIZE_PICMIP
</td>
<td> 2
</td>
<td> Apply picmip to the render target's width and height.
</td></tr>
<tr>
<td> RT_SIZE_HDR
</td>
<td> 3
</td>
<td> frame_buffer_width / 4
</td></tr>
<tr>
<td> RT_SIZE_FULL_FRAME_BUFFER
</td>
<td> 4
</td>
<td> Same size as frame buffer, or next lower power of 2 if we can't do that.
</td></tr>
<tr>
<td> RT_SIZE_OFFSCREEN
</td>
<td> 5
</td>
<td> Target of specified size, don't mess with dimensions
</td></tr>
<tr>
<td> RT_SIZE_FULL_FRAME_BUFFER_ROUNDED_UP
</td>
<td> 6
</td>
<td> Same size as the frame buffer, rounded up if necessary for systems that can't do non-power of two textures.
</td></tr>
<tr>
<td> RT_SIZE_REPLAY_SCREENSHOT
</td>
<td> 7
</td>
<td> Rounded down to power of 2, essentially
</td></tr>
<tr>
<td> RT_SIZE_LITERAL
</td>
<td> 8
</td>
<td> Use the size passed in. Don't clamp it to the frame buffer size. Really.
</td></tr>
<tr>
<td> RT_SIZE_LITERAL_PICMIP
</td>
<td> 9
</td>
<td>
</td></tr></table>
